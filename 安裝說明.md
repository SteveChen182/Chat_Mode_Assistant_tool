# Chat Mode Assistant — 新電腦安裝說明

## 前提條件

- Windows 10/11
- Python 3.10+（已加入 PATH）
- Intel Developer Toolkit CLI (`dt`) 已安裝並加入 PATH
- GNAI 帳號已設定完成（`dt gnai auth` 可正常使用）
- Google Chrome 瀏覽器

---

## 安裝步驟

### 1. Clone 專案

```powershell
git clone https://github.com/SteveChen182/Chat_Mode_Assistant_tool.git
cd Chat_Mode_Assistant_tool
```

### 2. 安裝 Python 依賴

```powershell
pip install pywinpty
```

或使用 requirements.txt：

```powershell
pip install -r bridge/requirements.txt
```

### 3. 載入 Chrome Extension

1. 開啟 Chrome，前往 `chrome://extensions/`
2. 右上角開啟「開發人員模式」
3. 點擊「載入未封裝項目」→ 選擇 `extension/` 資料夾
4. 記下產生的 **Extension ID**（32 字元小寫字串）

### 4. 設定 Native Messaging（自動啟動 Bridge）

```powershell
cd bridge
.\install_native_host.ps1 -ExtensionId <你的Extension ID>
```

此腳本會自動：
- 檢查 `dt` 和 `python` 是否可用
- 生成 `native_host.cmd`（指向正確的 Python 路徑）
- 生成 `com.chat_mode_assistant.bridge.json`（Native Messaging 設定檔）
- 寫入 Windows Registry 註冊 Native Messaging Host

### 5. 驗證

1. 重新啟動 Chrome
2. 點擊 Extension icon 開啟 Side Panel
3. 應自動連接 Bridge Server，狀態顯示 "Connected"

---

## 手動啟動 Bridge（不使用 Native Messaging）

如果不想設定自動啟動，可以手動執行：

```powershell
cd bridge
python bridge_server.py
```

Bridge 啟動後，Chrome Extension 會自動偵測並連線（Port 8776）。

---

## 常見問題

| 問題 | 原因 | 解決方式 |
|------|------|----------|
| 狀態顯示 "No Bridge" | Bridge Server 未啟動 | 執行 `python bridge_server.py` 或確認 Native Messaging 設定 |
| Native Messaging 啟動失敗 | 未執行 `install_native_host.ps1` | 重新執行 setup script |
| Session 無法建立 | `dt` 不在 PATH 中 | 確認 `dt version` 可執行 |
| Import 失敗 | pywinpty 未安裝 | `pip install pywinpty` |
| Extension ID 變更 | 重新載入 extension | 重新執行 `install_native_host.ps1` 填入新 ID |

---

## 環境變數（可選）

| 變數 | 預設值 | 說明 |
|------|--------|------|
| `BRIDGE_PORT` | 8776 | Bridge Server 監聽 Port |
| `BRIDGE_HOST` | 127.0.0.1 | Bridge Server 監聽 IP |
| `BRIDGE_ASSISTANT` | sighting_assistant | 預設 GNAI Assistant 名稱 |
| `BRIDGE_DT_PATH` | （空，自動搜尋） | 指定 dt CLI 路徑 |
| `BRIDGE_DEBUG` | 1 | 啟用 debug log |
