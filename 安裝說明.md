# Chat Mode Assistant — 新電腦安裝說明

## 前提條件

- Windows 10/11
- Python 3.10+（已加入 PATH）
- Intel Developer Toolkit CLI (`dt`) 已安裝並加入 PATH
- GNAI 帳號已設定完成（`dt gnai auth` 可正常使用）
- Google Chrome 瀏覽器

---

## 安裝步驟

### 1. Clone 專案

```powershell
git clone --recurse-submodules https://github.com/SteveChen182/Chat_Mode_Assistant_tool.git
cd Chat_Mode_Assistant_tool
```

> 如果已經 clone 但忘記加 `--recurse-submodules`，執行：
> ```powershell
> git submodule update --init --recursive
> ```

### 2. 安裝 Python 依賴 + 設定 Native Messaging（一步完成）

```powershell
cd bridge
.\install_native_host.ps1
```

此腳本會自動：
- 檢查 `dt` 和 `python` 是否可用
- 安裝所有 Python 依賴（`pip install -r requirements.txt`）
- 生成 `native_host.cmd`（指向正確的 Python 路徑）
- 生成 `com.chat_mode_assistant.bridge.json`（Native Messaging 設定檔）
- 寫入 Windows Registry 註冊 Native Messaging Host

### 3. 載入 Chrome Extension

1. 開啟 Chrome，前往 `chrome://extensions/`
2. 右上角開啟「開發人員模式」
3. 點擊「載入未封裝項目」→ 選擇 `extension/` 資料夾

### 4. 驗證

1. 重新啟動 Chrome
2. 點擊 Extension icon 開啟 Side Panel
3. 應自動連接 Bridge Server，狀態顯示 "Connected"

---

## 手動啟動 Bridge（不使用 Native Messaging）

如果不想設定自動啟動，可以手動執行：

```powershell
cd bridge
python bridge_server.py
```

Bridge 啟動後，Chrome Extension 會自動偵測並連線（Port 8776）。

## Check GFX/GOP Driver Regression 功能說明

GFX/GOP Driver Regression 功能已整合進 Chat Mode Assistant，不需要額外安裝。

### 使用方式

在 Side Panel 內，對話視窗頂端有兩個按鈕：

| 按鈕 | 功能 |
|------|------|
| **GFX** | 對當前 HSD 進行 GFX Driver Regression 分析 |
| **GOP** | 對當前 HSD 進行 GOP Firmware Regression 分析 |
| **H** | 開啟 History 面板（顯示過去的查詢結果）|
| **C** | 清除當前 HSD 的所有 Regression 記錄 |

### 查詢流程

```
點擊 GFX / GOP
  ↓
① 同一 HSD History lookup（by hsd_id）       ← 最快，直接顯示
  ↓（沒找到）
② Cross-HSD History lookup（by fail/pass 版本）← 其他 HSD 有相同版本直接借用
  ↓（沒找到）
③ Per-version Build Cache（組裝個別 build）  ← 跳過 QuickBuild 登入
  ↓（沒找到）
④ QuickBuild 登入後重新查詢
```

### 第一次使用前提

- 需要有 QuickBuild 帳號（Intel 內部網路）
- 第一次查詢每個版本組合時，會彈出 QB 登入視窗
- 查詢結果會自動存入本機 History Cache，後續相同版本不需重新登入

### History Cache 存放位置

```
bridge/
├── driver_history_gfx.json   ← GFX 查詢紀錄
├── driver_history_gop.json   ← GOP 查詢紀錄
├── build_cache_gfx.json      ← GFX per-version build 快取
└── build_cache_gop.json      ← GOP per-version build 快取
```

> 這些 JSON 檔案已列在 `.gitignore`，不會 commit 進 repo。

## 常見問題

| 問題 | 原因 | 解決方式 |
|------|------|----------|
| 狀態顯示 "No Bridge" | Bridge Server 未啟動 | 執行 `python bridge_server.py` 或確認 Native Messaging 設定 |
| Native Messaging 啟動失敗 | 未執行 `install_native_host.ps1` | 重新執行 setup script |
| Session 無法建立 | `dt` 不在 PATH 中 | 確認 `dt version` 可執行 |
| Import 失敗 | pywinpty 未安裝 | `pip install pywinpty` |
| Extension ID 變更 | 重新載入 extension | 重新執行 `install_native_host.ps1` 填入新 ID |

## 環境變數（可選）

| 變數 | 預設值 | 說明 |
|------|--------|------|
| `BRIDGE_PORT` | 8776 | Bridge Server 監聽 Port |
| `BRIDGE_HOST` | 127.0.0.1 | Bridge Server 監聽 IP |
| `BRIDGE_ASSISTANT` | sighting_assistant | 預設 GNAI Assistant 名稱 |
| `BRIDGE_DT_PATH` | （空，自動搜尋） | 指定 dt CLI 路徑 |
| `BRIDGE_DEBUG` | 1 | 啟用 debug log |
