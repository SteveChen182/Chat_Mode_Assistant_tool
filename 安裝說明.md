# Chat Mode Assistant — 新電腦安裝說明

## 前提條件

| 項目 | 說明 |
|------|------|
| **Windows 10/11** | 僅支援 Windows |
| **Intel Developer Toolkit CLI (`dt`)** | 需加入 PATH，且 `dt gnai auth` 已完成認證 |
| **GNAI 帳號** | 需有 `sighting_assistant`、`displaydebugger`、`sherlog` 存取權限 |
| **Google Chrome** | 用於載入 Extension |
| **Python** | **一般使用者不需要**；僅開發人員建置 installer 時才需要 |

---

## 一般使用者安裝（使用 .exe 安裝檔）

### 步驟 0（建議）：執行環境檢查

在安裝前，可先執行 `check_env.exe` 確認所有必要工具已正確安裝：

> `check_env.exe` 包含在安裝檔中，也可以單獨從開發人員處取得。

檢查項目：

| 項目 | 說明 |
|------|------|
| Intel `dt` CLI | 是否在 PATH 中 |
| GNAI 認證 | `dt gnai auth` 是否已完成 |
| sighting | Toolkit 安裝 + Assistant 存取權限 |
| displaydebugger | Toolkit 安裝 + Assistant 存取權限 |
| sherlog | Toolkit 安裝 + Assistant 存取權限 |

每項顯示格式：`Toolkit: installed  |  Assistant: accessible`

> ⚠️ Assistant 存取顯示 **⚠（黃色）** 表示無法自動驗證（dt 需要 TTY），顯示 **✗（紅色）** 表示 Toolkit 未安裝或確定無存取權限。

**自動安裝缺少的 Toolkit**：當某個 Toolkit 顯示「not installed」時，該列下方會出現 **▶ Install** 按鈕，點擊後會開啟 PowerShell 視窗自動執行對應的 register 指令，安裝完成並關閉視窗後會自動重新檢查。對應指令如下：

| Toolkit | 安裝指令 |
|---------|----------|
| sighting | `dt gnai toolkits register intel-sandbox/SightingAssistantTool` |
| displaydebugger | `dt gnai toolkits register intel-sandbox/displaydebugger` |
| sherlog | `dt gnai toolkits register intel-innersource/drivers.gpu.core.sherlog-toolkit` |

所有項目顯示 ✓ 後再繼續安裝步驟。

### 步驟 1：執行安裝檔

執行 `Chat_Mode_Assistant_Setup.exe`，依照精靈指示完成安裝。

- 安裝路徑：`%LOCALAPPDATA%\ChatModeAssistant\`（不需要系統管理員權限）
- 自動完成：複製 `bridge_server.exe`、`native_host.exe`、`extension/` 資料夾，並寫入 Registry 設定 Native Messaging

### 步驟 2：載入 Chrome Extension

安裝完成後，精靈會自動開啟 extension 資料夾：

1. 開啟 Chrome，前往 `chrome://extensions/`
2. 右上角開啟「**開發人員模式**」
3. 點擊「**載入未封裝項目**」→ 選擇安裝目錄內的 `extension\` 資料夾
   - 預設路徑：`%LOCALAPPDATA%\ChatModeAssistant\extension\`
4. 載入後 **Extension ID** 固定為 `pmbnnkfhdkommfpphknjpppmlmbihomi`

> Extension 的 `manifest.json` 內含固定的 `key`，因此 Extension ID 永遠相同，且已預先寫入 Native Messaging 設定，**不需要額外設定 Extension ID**。

### 步驟 3：驗證

1. 重新啟動 Chrome
2. 點擊 Extension icon 開啟 Side Panel
3. 狀態應顯示 "Connected"

## 架構概覽

```
Chrome Extension (Side Panel)
   │  Native Messaging（啟動）
   ▼
native_host.exe ──啟動──► bridge_server.exe（HTTP/SSE，Port 8776）
                              │  pywinpty / ConPTY
                              ▼
                          dt gnai chat --json --assistant <name>
```

- **Extension**：Side Panel UI（`background.js` service worker 透過 Native Messaging 啟動 Bridge，再以 HTTP/SSE 連線）
- **native_host.exe**：Chrome 呼叫的 Native Messaging Host，負責啟動 Bridge 並回傳 Port
- **bridge_server.exe**：HTTP/SSE 伺服器，以 ConPTY 管理 `dt gnai chat` 程序，提供 session、file dialog、driver history、QuickBuild 等端點

### 三種模式 / Assistant

| 模式 | 使用的 Assistant | Toolkit | 用途 |
|------|------------------|---------|------|
| **Chat Mode**（預設） | `sighting_assistant` | sighting | HSD 分析、與程式碼變更關聯 |
| **Log Analysis Mode** | `displaydebugger` | displaydebugger | Log 分析、Display driver 除錯 |
| **GFX/GOP Regression** | `sherlog`（+ QuickBuild） | sherlog | GPU driver / GOP firmware regression 分析 |

## 開發人員：建置安裝檔

> 一般使用者不需要此步驟，僅在需要修改程式碼後重新打包時使用。

### 額外需求

| 項目 | 說明 |
|------|------|
| **Python 3.10+** | 需加入 PATH（不可用 Windows Store 版）|
| **pywinpty** | `pip install pywinpty` |
| **PyInstaller** | `pip install pyinstaller`（首次執行 build.ps1 會自動安裝）|
| **Inno Setup 6** | 下載：https://jrsoftware.org/isinfo.php |

### 建置步驟

從專案根目錄執行：

```powershell
.\installer\build.ps1
```

build.ps1 會自動：
1. 確認 Python、pywinpty、PyInstaller 已安裝
2. 以 PyInstaller 將 `bridge_server.py`、`native_host.py`、`check_env.py` 打包成獨立 `.exe`
3. 呼叫 Inno Setup 產生安裝檔（內含 `bridge_server.exe`、`native_host.exe`、`extension/`）
4. 將 `check_env.exe` 複製到輸出目錄（獨立工具，不打包進安裝檔）

輸出：
- `C:\Intel\Chat_Mode_Assistant_Setup.exe` — 主安裝檔
- `C:\Intel\check_env.exe` — 獨立環境檢查工具

### Extension ID（setup.iss）

Extension 的 `manifest.json` 內含固定 `key`，因此 Extension ID 恆為：

```ini
#define ExtensionId "pmbnnkfhdkommfpphknjpppmlmbihomi"
```

此 ID 已寫死在 `setup.iss`，安裝時用於產生 `nm_manifest.json` 的 `allowed_origins`。除非更換 `manifest.json` 的 `key`，否則不需修改。

## 手動啟動 Bridge（開發 / 除錯用）

不想使用 Native Messaging 自動啟動時，可直接執行：

```powershell
# 從 source 執行（需要 Python）
cd bridge
python bridge_server.py

# 或執行打包好的 exe（不需要 Python）
%LOCALAPPDATA%\ChatModeAssistant\bridge_server.exe
```

Bridge 啟動後，Chrome Extension 會自動偵測並連線（Port 8776）。

## 已知問題與解決方法

| 問題 | 原因 | 解決方法 |
|------|------|----------|
| 狀態顯示 "No Bridge" | Native Messaging 未正確設定 | 重新執行安裝檔，確認 Registry 中已註冊 `com.chat_mode_assistant.bridge` |
| Extension 無法連線 | Extension 未正確載入，或 ID 不符 | 確認 Extension ID 為 `pmbnnkfhdkommfpppmlmbihomi`（由 manifest `key` 固定） |
| Session 無法建立 | `dt` 不在 PATH 或未完成 `dt gnai auth` | 先執行 `check_env.exe` 檢查，確認 `dt gnai auth` 可正常執行 |
| Toolkit 未安裝 | 尚未 register 對應 toolkit | 執行 `check_env.exe` 並點選 **▶ Install**，或手動執行 `dt gnai toolkits register ...` |
| 安裝時提示需要管理員 | 系統策略限制 | 選擇「Install for current user only」繞過 |

## Check GFX/GOP Driver Regression 功能說明

GFX/GOP Driver Regression 功能已整合進 Chat Mode Assistant，不需要額外安裝。

### 使用方式

在 Side Panel 內，對話視窗頂端有兩個按鈕：

| 按鈕 | 功能 |
|------|------|
| **GFX** | 對當前 HSD 進行 GFX Driver Regression 分析 |
| **GOP** | 對當前 HSD 進行 GOP Firmware Regression 分析 |
| **H** | 開啟 History 面板（顯示過去的查詢結果）|
| **C** | 清除當前 HSD 的所有 Regression 記錄 |

### 查詢流程

```
點擊 GFX / GOP
  ↓
① 同一 HSD History lookup（by hsd_id）       ← 最快，直接顯示
  ↓（沒找到）
② Cross-HSD History lookup（by fail/pass 版本）← 其他 HSD 有相同版本直接借用
  ↓（沒找到）
③ Per-version Build Cache（組裝個別 build）  ← 跳過 QuickBuild 登入
  ↓（沒找到）
④ QuickBuild 登入後重新查詢
```

### 第一次使用前提

- 需要有 QuickBuild 帳號（Intel 內部網路）
- 第一次查詢每個版本組合時，會彈出 QB 登入視窗
- 查詢結果會自動存入本機 History Cache，後續相同版本不需重新登入

### History Cache 存放位置

安裝後 cache 存放於安裝目錄：

```
%LOCALAPPDATA%\ChatModeAssistant\
├── driver_history_gfx.json   ← GFX 查詢紀錄
├── driver_history_gop.json   ← GOP 查詢紀錄
├── build_cache_gfx.json      ← GFX per-version build 快取
└── build_cache_gop.json      ← GOP per-version build 快取
```

## 環境變數（可選）

| 變數 | 預設值 | 說明 |
|------|--------|------|
| `BRIDGE_PORT` | 8776 | Bridge Server 監聽 Port |
| `BRIDGE_HOST` | 127.0.0.1 | Bridge Server 監聽 IP |

| `BRIDGE_DEBUG` | 1 | 啟用 debug log |

## 安裝目錄結構（安裝後）

```
%LOCALAPPDATA%\ChatModeAssistant\
├── bridge_server.exe     ← Bridge Server（獨立執行，不需 Python）
├── native_host.exe       ← Native Messaging Host（Chrome 自動呼叫）
├── nm_manifest.json      ← Native Messaging 設定（安裝時自動生成）
└── extension\            ← Chrome Extension（Load unpacked 此資料夾）
    ├── manifest.json
    ├── background.js
    ├── sidepanel.html
    ├── sidepanel.js
    └── icons\
```

> `check_env.exe` 不隨安裝檔安裝，為獨立工具，請在安裝前單獨執行。

## 原始碼結構（開發人員）

```
Chat_Mode_Assistant_tool/
├── extension/            ← Chrome Extension 原始碼
│   ├── manifest.json
│   ├── background.js
│   ├── sidepanel.html
│   ├── sidepanel.js
│   └── icons/
├── bridge/               ← Bridge Server 原始碼
│   ├── bridge_server.py
│   ├── native_host.py
│   └── requirements.txt
└── installer/            ← 安裝檔建置工具

    ├── setup.iss         ← Inno Setup 設定
    ├── check_env.py      ← 環境檢查工具原始碼
    ├── configure.py      ← Extension ID 設定工具原始碼（目前未隨安裝檔散佈）
    └── dist/             ← PyInstaller 輸出（build.ps1 產生）
```
