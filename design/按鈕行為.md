# 按鈕交互邏輯整理

## 1. New (`btn-new`)

| 條件 | 行為 |
|------|------|
| session 為空（`sessionMessages.length === 0` 且無 `activeHsdId`） | 直接執行 `startNewSession()`，不彈確認 |
| session 有內容 | 彈出 Modal 確認「Leave current session and start a new one?」 |
| 使用者確認 | 執行 `startNewSession()` |
| 使用者取消 | 不做事 |

**`startNewSession()` 做的事：**
- 儲存目前 session 到 history
- 清空 chatArea、quickActions、heroCta、postAnalysisPanel
- 重設狀態：`activeHsdId`, `activeHsdTitle`, `activeConversationId`, `sessionMessages` 全歸零
- 顯示 onboarding 歡迎畫面
- 在 `sessions` 陣列前面插入一筆空 session，超過 `MAX_SESSIONS(6)` 則截斷
- 發 `{ action: "start_session" }` 給 background.js

**啟用/停用狀態：** 始終 enabled，不受 `setInputEnabled()` 控制。

---

## 2. Stop (`btn-stop`)

| 條件 | 行為 |
|------|------|
| 任何時候按下 | 直接發 `{ action: "stop_session" }` 給 background.js，無確認 |

**後續效果（來自 `session_stopped` 事件回應）：**
- 狀態改為 `disconnected / Offline`
- 移除 tool indicator、typing indicator
- 重設 streaming 狀態
- 顯示系統訊息「Session ended.」
- 顯示 toast「Session ended」
- 停用輸入（`setInputEnabled(false)`）

**啟用/停用狀態：** 始終 enabled，不受 `setInputEnabled()` 控制。

---

## 3. Import HSD (`btn-import`)

| 條件 | 行為 |
|------|------|
| 按下時 | 呼叫 `importHsdFromWebpage()` |
| 當前 tab URL 無 HSD ID | 顯示系統訊息提示使用者先開啟 HSD-ES 頁面 |
| 已有不同的 `activeHsdId` | 彈 Modal 確認是否切換到新 HSD |
| 確認/首次匯入 | 設定 `activeHsdId`、更新 header title、生成 conversation ID、取頁面標題設為 subtitle、顯示「Imported HSD ID: xxx」、toast 通知、設 `hsdImported = true`、顯示大按鈕 hero CTA「🚀 Analyze HSD xxx」 |
| 使用者按 hero CTA | 發送 `"{hsdId} skip any attachment check"` 作為第一則訊息，且因 `hsdImported` 為 true 會自動加上 `[HSD xxxxx]` 前綴 |

**啟用/停用狀態：** 受 `setInputEnabled()` 控制 — AI 回應中會被 disabled，ready 後 enabled。

---

## 4. History (`btn-history`) — 顯示為 ↺ 圖示

| 條件 | 行為 |
|------|------|
| menu 已展開 | 收起 menu（toggle 關閉） |
| menu 未展開 | 先 `saveCurrentSession()`，然後渲染 history menu |
| menu 內容 | 顯示 `sessions[1:]`（跳過當前 active session），每筆顯示 HSD ID、標題、CID、時間 |
| 點擊某筆 history item | 關閉 menu → `switchToSession(index)`：儲存當前、將目標 session 移到陣列最前面載入、重建 chatArea |
| 點刪除 🗑️ | 彈 Modal 確認 → 刪除該筆 session → 刷新 menu |
| 點擊 menu 外部 / 按 Escape | 關閉 menu |

**啟用/停用狀態：** 受 `setInputEnabled()` 控制 — AI 回應中會被 disabled，ready 後 enabled。

---

## 整體 `setInputEnabled(enabled)` 影響範圍

```
受控：inputEl, sendBtn, btnImport, btnHistory
不受控：btnNew, btnStop（永遠 enabled）
```

這代表使用者在 AI streaming 回應中仍可以按 New 或 Stop，但不能按 Import HSD 或 History。
